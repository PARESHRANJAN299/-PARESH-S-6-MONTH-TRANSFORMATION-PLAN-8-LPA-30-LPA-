You have a table with 5 columns. Your team should only see 3. Two ways to do it:
Original table — employees
All 5 columns:
emp_id name department salary ← sensitive bank_account ← sensitive
WAY 1 — Create a View
Simple & most common
CREATE VIEW team_view AS
SELECT emp_id, name, department
FROM employees;
Then:
GRANT SELECT ON VIEW team_view
TO `team@company.com`;
Team only sees:
emp_id name department
Salary and bank_account don't even exist in the view. Team has no idea they exist.
WAY 2 — Column Masking
Advanced & dynamic
ALTER TABLE employees
ALTER COLUMN salary
SET MASK mask_salary_fn;
Team queries the real table but sees:
emp_id name department salary → **** bank → ****
They CAN see the column exists, but value is hidden. Admin still sees real value.
When to use which?
View Team should not even know the hidden columns exist. Simpler. Most teams use this.
Masking Same table, but some users see real value, others see ****. Good when admin and analyst use the same table.
